旅行商问题的遗传算法：
交配方法：刚开始用常规交配法，然后发现代价（旅行商走过的距离）太长了。
于是选用一种比较类似贪心算法的交配方法，即两个父代交配的时候得到的子代的路径中每两个城市的距离都是可以从父代那里继承的最短路径
（比如两个父代一个是1→3→2→4→1，一个是1→2→3→4→1。从第2个城市开始遍历，假设1→2的距离短于1→3的距离，子代的第一个城市就选择1→2，依次遍历）
由于新的交配法每次交配得到的子代都是最优的，所以新的交配方法得到的代价比常规交配法小了3-4倍左右
（常规交配法：即找定一个交配位，交配位之前的的基因选自父代。交配位之后的基因在另一个父代中按顺序选取没有出现过的基因。）
变异方法：基于次序的变异规则（随机产生两个变异位，然后交换这两个变异位上的基因）
保留（淘汰规则）：先将种群（每个旅行方法）按代价排序，保留保留概率PS的种群法，将后面被淘汰的种群用排名在前面的种群，然后交配的时候注意两个相同的种群不交配即可。
